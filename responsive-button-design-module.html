Proper button sizing is crucial for creating accessible, user-friendly interfaces. These guidelines ensure your buttons work well across all devices and meet accessibility standards.

Why Button Size Matters

Button size directly impacts usability and accessibility:

  • Touch Targets: Mobile users need adequately sized buttons (minimum 44px √ó 44px per WCAG 2.1 Level AAA)
  • Visual Hierarchy: Different sizes communicate importance and guide user attention
  • Consistency: Standardized sizes create a cohesive, professional design system
  • Accessibility: Proper sizing helps users with motor impairments or visual limitations

Accessibility Guidelines

Following these standards ensures your buttons meet accessibility requirements:

Touch Target Standards
  • WCAG 2.1 Level AAA: Minimum 44px √ó 44px
  • WCAG 2.2 Level AA: Minimum 24px √ó 24px
  • Apple HIG: Minimum 44pt (~44px)
  • Material Design: Minimum 48dp (~48px)
  • Industry Best Practice: 48px √ó 48px for mobile

Spacing Recommendations

  • Minimum: 8px between adjacent buttons
  • Recommended: 12-16px for comfortable interaction
  • Mobile: 16px+ to prevent mis-taps

Responsive Strategy

Adapt button sizes based on device and user context:

/* Mobile-first: Start with comfortable touch targets */
.btn {
  height: 48px;
  padding: 0 20px;
  font-size: 16px;
}

/* Tablet and up: More compact */
@media (min-width: 768px) {
  .btn {
    height: 40px;
    padding: 0 16px;
    font-size: 14px;
  }
}

/* Desktop: Maximum density */
@media (min-width: 1024px) {
  .btn-secondary {
    height: 36px;
    font-size: 13px;
  }
}

Key Takeaways

  • Use small buttons (32-36px) for secondary actions and compact interfaces
  • Use medium buttons (40-48px) for most primary and secondary actions
  • Use large buttons (52-56px) for CTAs and mobile-first designs
  • Always meet the 44px √ó 44px minimum for touch targets (WCAG AAA)
  • Provide adequate spacing between buttons (12-16px recommended)
  • Use a mobile-first approach and scale down for desktop
